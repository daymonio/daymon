Privacy Policy

Daymon is a local-first, open-source desktop application. Your data stays on your machine. This policy explains what data exists, where it lives, and what the website collects.

Effective date: February 16, 2026

1. The Daymon Application

  • Daymon runs entirely on your local machine. There is no Daymon cloud backend, no user accounts, and no authentication.
  • All data — tasks, memories, workers, run history — is stored in a local SQLite database on your computer.
  • Daymon does not transmit any of your data to Daymon servers because there are no Daymon servers.
  • Daymon offers an optional, opt-in setting to share anonymous crash reports and diagnostics (app version, OS, aggregate counts). This is disabled by default. No personal information, task content, or memory data is ever transmitted.
  • You can inspect, export, or delete your data at any time. It is a single SQLite file on your machine.

2. Third-Party Services

  • Daymon connects to Claude CLI (by Anthropic) to execute tasks. When a task runs, the prompt and any context are sent to Anthropic's API through Claude CLI. This is governed by Anthropic's privacy policy and terms of service.
  • Daymon does not control, intercept, or store the data transmitted between Claude CLI and Anthropic's servers.
  • Local embeddings are computed using Hugging Face Transformers running entirely on your machine. No data is sent to Hugging Face.

3. The Daymon Website

  • The website at daymon.io uses Google Analytics to understand visitor traffic. Google Analytics may use cookies to collect anonymous usage data such as page views, referral sources, and browser type.
  • The website also uses Google Ads conversion tracking on the homepage to measure download button clicks.
  • The website is hosted on GitHub Pages. GitHub may collect server logs (IP addresses, request timestamps). See GitHub's privacy statement.
  • The website makes a client-side request to the GitHub API to display the repository star count. This request is subject to GitHub's privacy policy.

4. Cookies

  • The Daymon application does not use cookies.
  • The Daymon website may set cookies through Google Analytics and Google Ads conversion tracking. These are third-party cookies managed by Google. You can opt out of Google Analytics by installing the Google Analytics Opt-out Browser Add-on.

5. Data You Create

  • Memories, tasks, workers, and run history you create in Daymon are stored locally in a SQLite database.
  • You have full control over this data. You can view, modify, or delete it at any time.
  • Daymon is open source (MIT license) — you can audit exactly what the application does with your data by reviewing the source code.

6. Children's Privacy

Daymon is not directed at children under 13. We do not knowingly collect personal information from children.

7. Changes to This Policy

We may update this privacy policy from time to time. Changes will be posted on this page with an updated effective date. Since Daymon is open source, all changes are visible in the public repository.

8. Contact

If you have questions about this privacy policy, email us at hello@daymon.io, open an issue on GitHub, or reach out on X (@Daymon_io).
